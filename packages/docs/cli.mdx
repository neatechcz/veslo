Use the CLI stack when you want automation without the desktop app.

## Main commands

- `veslo`: headless orchestrator (OpenCode + Veslo server + router)
- `veslo-server`: filesystem-backed remote API server
- `opencode-router`: Slack/Telegram bridge with directory routing

## veslo (recommended)

```bash
npm install -g veslo
veslo start --workspace /path/to/workspace --approval auto
```

Useful follow-ups:

```bash
veslo status --veslo-url http://127.0.0.1:8787 --opencode-url http://127.0.0.1:4096
veslo approvals list --veslo-url http://127.0.0.1:8787 --host-token <token>
```

## veslo-server

```bash
npm install -g veslo-server
veslo-server --workspace /path/to/workspace --approval auto
```

Key env vars:

- `VESLO_TOKEN`
- `VESLO_HOST_TOKEN`
- `VESLO_APPROVAL_MODE`

## opencode-router

```bash
npm install -g opencode-router
opencode-router start
```

Pair it with bindings for deterministic routing by peer + directory.
