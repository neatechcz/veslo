Veslo Multi-Tenant Cloud MVP

MVP Only This document contains only V1 functionality required to ship multi-tenancy.

1. MVP Scope

Area MVP Decision
Tenant boundaryAll data entities are scoped by org_id; every request runs with active org context.
Rolesplatform_admin, org_owner, org_member.
UI termUse Projects in UI; backend may keep worker naming for compatibility.
Project creationAny org user can create projects in V1.
Platform settings visibilityOnly platform admins can view/edit platform settings.
Organization settingsOwners can manage org settings (join policy, domain policy, model profile selection).
BillingExactly one org subscription; owner-managed; self-serve + admin-invoiced modes.
Admin trial controlAdmin can set or extend trial end to any date/time.

2. Domain Onboarding (MVP)

Condition User Action Options Result
No org exists for domainCreate orgUser becomes first owner.
Org exists for domain + creation allowedCreate new org or request join existingBoth options visible.
Org exists + owner set domain join-onlyRequest join existing onlyNew org option hidden/disabled.
Join requires owner approvalSubmit join requestMembership activated only after owner approval.
Public domain emailCreate personal org by default, or join via invite/requestNo hard dependency on complete public-domain list.

3. Permissions (MVP)

Capability Member Owner Platform Admin
Create projectYesYesYes

Manage org members/rolesNoYesYes
View/edit org settingsRead/use only (configurable)YesYes

Suspend org executionNoNoYes (exceptional only)

All admin interventions must create immutable audit logs with reason and scope.

6. MVP Data Model

Table Purpose
platform_roleGlobal admin grants.
orgOrganization root and policy flags.
org_domainDomain mapping and domain policy.
org_membershipOwner/member roles per org.
org_join_requestJoin approval workflow.
org_settingsOrg-level settings including model profile.
org_subscriptionOne subscription per org plus trial end.
org_billing_requestInvoice approval flow.
audit_eventSecurity and governance audit trail.

7. Not in MVP

Deferred Feature Reason
Global artifact marketplace and approvalsNot required to ship multi-tenancy foundation.
Member contribution and org-to-global promotion flowsSecond-stage capability after tenant baseline.
AI project audits and auto-stopFuture policy layer.
Owner-inaccessible private member projectsFuture privacy model; keep design space open.
