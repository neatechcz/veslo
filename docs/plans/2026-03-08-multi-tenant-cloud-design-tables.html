2. Domain Login Decision Matrix

Condition at Login User Choices Owner Policy Influence Outcome
No org mapped to domain Create new org N/A Org created, user becomes owner.
Org exists for domain, creation allowed Create new org or request join existing Owner can keep creation allowed. User picks either path.
Org exists for domain, creation locked Join existing only Owner sets domain to join-only. User cannot create another org for that domain.
Join request submitted Wait for approval Owner approval required when policy says so. Membership becomes active after owner approval.
Domain auto-join enabled Join immediately Owner can disable auto-join anytime. User becomes member without manual approval.
Public domain user (gmail, etc.) Create personal org by default; join via invite/request Owner invite/join policy still applies. No blind domain-based auto-grouping.

3. Billing State Model (Org-Level)

Billing State Entry Trigger Who Can Change System Effect
trialing New org trial or admin custom trial assignment. Admin / Owner (request) Trial entitlements active.
active_self_serve Successful Stripe/Polar checkout. Owner / Admin Full paid access.
admin_invoiced_pending Owner requests invoice mode. Admin Pending approval; restricted as configured.
active_admin_invoiced Admin approves invoiced billing. Admin (Owner visible) Full access without in-app card requirement.
past_due Invoice overdue or card charge failed. Owner / Admin Grace period + warnings.
suspended Past due exceeds grace or admin suspension. Admin Cloud project creation blocked; restricted runtime.
canceled Owner/admin cancellation. Owner / Admin Access ends at period end.

6. Data Model Sketch

Table Purpose Critical Keys / Constraints
platform_role Global admin roles. user_id unique per role grant.
org (extend) Tenant root and policy config. id, slug unique, join_policy, billing_mode, allow_new_org_on_domain_match.
org_domain Email domain to org mapping + verification state. domain indexed (not always unique), org_id foreign key, allow_join, require_owner_approval.
org_membership (extend) Membership and role assignment. Unique (org_id, user_id).
org_invite Invite-only onboarding flow. token unique, expires_at.
org_join_request Owner-managed domain join requests. Unique open request per (org_id, user_id).
org_subscription Single subscription record per org. Unique org_id, provider ids indexed, trial_ends_at admin-settable.
org_billing_request Invoice approval workflow. org_id, status, approved_by_admin_id.
org_settings Organization-configurable behavior and model preferences. org_id unique, model/provider profile fields, policy toggles.
platform_settings Global infrastructure and provider credentials. Admin-only table; no org read access.
artifact Unified item for skill/MCP/automation. kind, scope (private_user/private_org/org/global), org_id nullable for global.
artifact_release Approval state transitions. artifact_id, state, immutable transition log.
artifact_distribution Rollout targeting and enable flags. artifact_id, target all_orgs or specific org_id.
audit_event (existing/extend) Security, billing, sharing auditability. Indexed org_id, actor, action, timestamp.

8. Phased Implementation Plan (Parts)

Phase Goal Minimum Deliverables Out of Scope (for that phase)
Phase 1: Tenant & Role Foundation Separate platform admin from org roles and enforce org context. Admin role table, active-org context in session/API, org-guard middleware, query hardening by org_id. Global catalog publishing UI.
Phase 2: Domain Join/Create Flows Implement domain-based decision matrix and owner-controlled join policy. Domain decision endpoint, create-vs-join UI flow, join request + owner approval flow, domain lock controls. Advanced DNS verification automation.
Phase 3: Org Billing Control Move billing to org-level ownership with admin invoice path. org_subscription, invoice approval workflow, admin custom trial duration control, owner billing screens. Complex finance reporting.
Phase 4: Project UI Migration Rename "Workers" to "Projects" in UI and allow creation by all org users. UI copy migration, permissions update for member project creation, backward-compatible backend naming. Backend entity rename from worker to project.
Phase 5: Artifact Sharing Overlay Add contribution, org publish, and global promotion flow. Private defaults, owner approval path, admin global approval gate, audit trail for publish actions. Marketplace/revenue features.

Source context: current cloud control plane under services/den, with existing org/worker baseline and extension points for true multi-org tenant operations.
